Sobre mí

Una historia
poco convencional

Mi camino hacia la tecnología no fue en línea recta — y eso es lo que más valor me ha dado. Durante más de 8 años trabajé simultáneamente como Auxiliar Contable y Encargada de Soporte de TI en una cooperativa, gestionando infraestructura tecnológica para 50 empleados en 4 departamentos.

Esa dualidad me enseñó algo que pocos QA traen al equipo: entender el negocio desde adentro y la tecnología que lo sostiene. Hoy me especializo en QA Engineering a través del bootcamp de TripleTen, completando proyectos prácticos en pruebas manuales, diseño de casos de prueba y pruebas de API.

También soy fundadora de Proyecto PC Rosa 🌸, una iniciativa social donde imparto talleres de cómputo para niños y adultos en San Juan de los Lagos — porque creo que la tecnología debe ser accesible para todos.

de QA

Proyectos desarrollados durante el bootcamp de QA Engineering en TripleTen, aplicando técnicas reales de aseguramiento de calidad.

Sprint 1 · TripleTen 2026

Introducción a QA Engineering

Contexto

Primer acercamiento a las pruebas manuales de software, aprendiendo a identificar errores en nuevas funcionalidades y cambios de sistema.

Análisis

Realicé pruebas de humo y regresión para verificar que los nuevos cambios no introdujeran errores en funcionalidades existentes. Documenté cada defecto encontrado con pasos de reproducción claros.

Conclusiones

Las pruebas de regresión son esenciales para mantener la estabilidad del software. Un buen reporte de bug acelera significativamente la corrección por parte del equipo de desarrollo.

Pruebas manuales Smoke Testing Regression Testing Bug Reporting
Sprint 2 · TripleTen 2026

Diseño de Pruebas

Contexto

Diseño de pruebas para un servicio que genera rutas para distintos tipos de transporte, analizando requisitos y creando casos de prueba estructurados.

Análisis

Analicé los requisitos funcionales del servicio e identifiqué escenarios críticos usando técnicas de partición de equivalencia. Creé listas de comprobación y casos de prueba cubriendo flujos positivos y negativos.

Conclusiones

Un buen análisis de requisitos antes de probar ahorra tiempo y reduce errores. La cobertura de casos límite es tan importante como los flujos principales.

Análisis de requisitos Test Case Design Checklists
Sprint 3 · TripleTen 2026

Pruebas de Aplicaciones Web

Contexto

Pruebas completas de una aplicación web: preparación de documentación, ejecución de tests y reporte de bugs en un entorno real de QA.

Análisis

Ejecuté pruebas de UI identificando defectos visuales y funcionales. Usé JIRA para el seguimiento completo del ciclo de vida de cada bug, desde su detección hasta el cierre.

Conclusiones

El seguimiento estructurado de bugs con herramientas como JIRA mejora la comunicación entre QA y desarrollo, reduciendo el tiempo de resolución de incidencias.

UI Testing JIRA Chrome DevTools Bug Tracking
Sprint 4 · TripleTen 2026

Pruebas de API

Contexto

Pruebas manuales de API REST, verificando que cada endpoint respondiera correctamente e identificando comportamientos inesperados en las solicitudes HTTP.

Análisis

Usé Postman para ejecutar solicitudes GET, POST, PUT y DELETE, validando códigos de respuesta, estructura del JSON y manejo de errores. Inspeccioné el tráfico HTTP con herramientas de red del navegador.

Conclusiones

Las pruebas de API permiten detectar errores de integración antes de que afecten la interfaz de usuario. Validar tanto casos exitosos como errores es fundamental para una cobertura completa.

Postman API Testing HTTP Requests ApiDoc
